About Timeseries Pipeline

Project Overview

Timeseries Pipeline is a comprehensive suite of tools for time series analysis and forecasting, focusing on financial and econometric modeling. The project consists of four main components that work together to provide a complete solution.

Tech Stack

Frontend

A Django web app provides an intuitive UI for analysis. Why Django?

  • less boilerplate
  • faster prototyping
  • batteries-included (auth, forms, routing)
GitHub Repository

API

A FastAPI service handles the computational tasks. Why FastAPI?

  • Fast, easy to write
  • auto-docs with Swagger/OpenAPI
GitHub Repository
Docker Hub
GraphQL Playground

Database

A Postgresql database stores pipeline run metadata and model results. Why postgres?

  • scales
  • good Python libraries (psycopg2, SQLAlchemy)

Compute Package

Python package implements statistical models. Why package?

  • package once, use everywhere (API, CLI)
  • Easy to publish and integrate with CI/CD

Architecture

The components interact according to the following architecture:

sequenceDiagram actor User participant Frontend as Frontend participant API as API participant Package as Compute Package participant DB as PostgreSQL DB User->>Frontend: Submit analysis request Frontend->>API: Forward request parameters API->>Package: Execute computations Package-->>API: Return computation results API->>DB: Write results to database API-->>Frontend: Return processed results Frontend-->>User: Display visualization %% Direct API interaction User-->>API: Direct REST/GraphQL request API-->>User: Direct API response

This separation of concerns allows for:

  • Scalable architecture with specialized components
  • Independent development and deployment of each component
  • Reuse of the various components

Project Origins

This project is a modern reimplementation of my 2012 master's research on volatility spillover effects in global markets. Back then, I was convinced the Chinese real estate market would collapse and impact the rest of the world, similar to the 2008 Global Financial Crisis.

That collapse never happened. But the research still holds value, especially given today's context of post-COVID supply disruptions and increased tariff volatility.

Why reiterate on decade-old work? Two main reasons: First, I wanted to productionalize the code to modern standards, applying current best practices for software engineering and data science. Second, it was an opportunity to flex some intellectual muscle while building something that feels more relevant to today's volatile world.
